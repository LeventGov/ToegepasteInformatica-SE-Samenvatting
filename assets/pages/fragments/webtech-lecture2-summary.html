1. Functies in JavaScript

Functie Definitie Manieren

Er zijn verschillende manieren om functies te definiëren:

  • Functiedeclaratie: Een traditionele manier die gebruikmaakt van het function keyword en wordt gehost.
  • Functie-expressie: Een functie die aan een variabele wordt toegewezen en niet wordt gehost.
  • Arrow notation: Een compact alternatief voor de functie-expressie.

Arrow Functions (Pijlnotatie)

Arrow functions hebben specifieke regels voor syntax:

  • Parameters: Bij één parameter zijn haakjes optioneel; bij meerdere parameters zijn ze verplicht.
  • Body: Bij een simpele expressie is een return niet nodig. Voor meerdere regels zijn accolades {} en een expliciete return vereist.
  • Object literals: Om een object literal direct terug te geven, moeten er haakjes om het object staan: (params) => ({ foo: "a" })
  • Features: Ze ondersteunen rest parameters, default parameters en destructuring binnen de parameters.

IIFE (Immediately Invoked Function Expression)

  • Een IIFE is een functie die onmiddellijk wordt uitgevoerd zodra deze is gedefinieerd.
  • Ze worden vaak gebruikt om de globale scope niet te vervuilen en voor eenmalige initialisatiecode.

2. Higher Order Functions (HOF)

First-Class Functions

JavaScript behandelt functies als First-class Functions, wat betekent dat ze kunnen worden:

  • Toegewezen aan variabelen
  • Als argumenten kunnen worden doorgegeven
  • Door andere functies kunnen worden geretourneerd

Higher-Order Functions

Een Higher-order function doet minstens één van de volgende zaken:

  • Neemt één of meer functies als argumenten.
  • Retourneert een functie als resultaat.

Bekende HOF Voorbeelden

  • .forEach: Voert een functie uit voor elk element.
  • .map: Maakt een nieuwe array op basis van de resultaten van een functie.
  • .filter: Maakt een kopie van een array gefilterd op elementen die een test doorstaan.
  • .reduce: Voert een "reducer" callback uit op elk element om tot één waarde te komen.
  • .addEventListener: Stelt een functie in die wordt aangeroepen bij een specifiek event.

3. Lexical Scope & Closures

Lexical Scope

  • Lexical scoping bepaalt de beschikbaarheid van variabelen op basis van hun locatie in de broncode.
  • Geneste functies hebben altijd toegang tot variabelen die in hun buitenste scope (outer scope) zijn gedeclareerd.

Variable Shadowing

  • Shadowing treedt op wanneer een variabele in een binnenste scope dezelfde naam heeft als een variabele in een buitenste scope.
  • JavaScript kijkt altijd eerst in de meest binnenste scope voor de declaratie van een variabele.

Closure

  • Een closure is de combinatie van een functie met referenties naar de omringende status (de lexical environment).
  • Het geeft een binnenste functie toegang tot de scope van een buitenste functie, zelfs nadat de buitenste functie is uitgevoerd.
  • Closures worden gecreëerd telkens wanneer een functie wordt aangemaakt.

4. Object-Oriented Programming (OOP)

Het this Keyword

  • Methodes: this verwijst naar het object zelf.
  • Globale ruimte: Verwijst naar het window object (browser) of global (Node.js).
  • Strict mode / ESM Node: this is undefined.
  • Arrow functions: Behouden de this waarde van de omringende lexical context.

Call, Apply & Bind

Dit zijn methodes om expliciet de waarde van this in te stellen:

  • call(): Roept een functie aan met een opgegeven this en individuele argumenten.
  • apply(): Werkt hetzelfde als call, maar de argumenten worden als een array doorgegeven.
  • bind(): Creëert een nieuwe functie met een permanent ingestelde this waarde.

Klassen en Objecten (ES6)

Klassen in ES6 bieden een overzichtelijke syntax voor OOP:

  • Constructor: Een speciale methode die wordt aangeroepen bij het maken van een nieuw object met het new keyword.
  • Getters: Gebruik get voor berekende eigenschappen; deze worden aangeroepen als een gewone eigenschap op het object.
  • Private fields: Variabelen die beginnen met een # zijn privé en alleen toegankelijk binnen de klasse.

Overerving (Inheritance)

  • extends: Gebruikt om een subklasse te maken van een basisklasse.
  • super: Gebruikt om de constructor of methodes van de basisklasse (superclass) aan te roepen.

5. Sets, Maps & Symbols

Sets

  • Set: Slaat unieke waarden van elk type op.
  • Methodes: .add(), .has(), .clear(), .forEach()

Maps

  • Map: Houdt key-value paren bij en onthoudt de volgorde van invoeging.
  • Sleutels en waarden kunnen elk type zijn (objecten of primitieven).

Symbol & Object.freeze

  • Symbol: Een unieke, onveranderlijke primitieve waarde, vaak gebruikt voor unieke keys.
  • Object.freeze(): Bevriest een object zodat het niet meer aangepast kan worden. Dit is de basis voor het maken van Enums in JavaScript.
