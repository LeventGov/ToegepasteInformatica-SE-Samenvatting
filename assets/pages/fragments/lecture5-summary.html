1. DevOps doelen en mentaliteit

DevOps koppelt development en operations om sneller, betrouwbaarder en herhaalbaar te releasen.

  • CALMS: Culture, Automation, Lean, Measurement, Sharing.
  • Kernprincipes: Korte feedbackloops, alles versiebeheer (infra as code), automatiseren waar mogelijk, meten en bijsturen.

2. Code- en releaseflow (Laravel context)

  • Branching: feature branches → merge naar dev/main via pull request en review.
  • CI: composer install, lint/tests (phpunit/pest), security checks, build artifacts.
  • CD: versie taggen, artefact deployen naar test/staging → productie na goedkeuring.
  • Rollback: houd vorige artefacten en DB-backups beschikbaar.

3. Laravel stack kernconcepten

  • MVC: Routes → Controllers → Views; Models via Eloquent ORM.
  • Composer: dependency manager; lockfile vastzetten voor reproduceerbare builds.
  • Artisan: CLI voor serve, migrate, tinker, queue:work, schedule:run.
  • Testing: phpunit/pest; prefer tests in CI voordat er gedeployed wordt.

4. Configuratie en secrets

  • .env: bevat APP_ENV, APP_KEY, DB_*, MAIL_*, QUEUE_*, CACHE_*; nooit committen.
  • Prod: APP_DEBUG=false; config:cache en route:cache voor performance.
  • Secretsbeheer: gebruik vault/secret manager of CI/CD variabelen, niet hardcoded.

5. Runtime stack

  • Weblaag: Nginx/Apache als reverse proxy → PHP-FPM; document root is public/.
  • Queues & schedulers: queue workers via supervisor/systemd; cron roept elke minuut php artisan schedule:run aan.
  • Stateless deploys: uploads naar object storage (bv. S3) zodat pods/servers stateless blijven.

6. Database en migrations

  • Migrations: versieer DB-schema; migrate en rollback voor evoluties.
  • Seeders: vullen testdata; handig voor dev/staging.
  • Eloquent: models met relationships (hasOne/hasMany/belongsToMany) + query builder.
  • Backups & migratie-venster: plan onderhoud, test schema changes in staging, neem backup voor productie-run.

7. Deployment checklist

  • composer install --no-dev --optimize-autoloader; npm ci && npm run build indien frontend assets.
  • php artisan config:cache, route:cache, view:cache voor snellere boot.
  • Controleer permissies op storage/ en bootstrap/cache.
  • Healthchecks: HTTP 200 op /health of vergelijkbaar; monitor logs en metrics (APM).
