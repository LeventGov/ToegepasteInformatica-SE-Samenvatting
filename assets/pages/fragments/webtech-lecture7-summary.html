1. Introductie en Geschiedenis

JavaScript werd in 1995 in tien dagen gemaakt door Brendan Eich voor Netscape als lichte scripttaal voor browsers. Naarmate webapps groeiden, werden beperkingen zichtbaar zoals het ontbreken van sterke typering, waardoor runtime-fouten laat opdoken.

TypeScript is een superset van JavaScript, ontwikkeld door Microsoft. Het voegt optionele statische types toe en wordt getranspileerd naar gewone JavaScript zodat elke browser het kan draaien.

  • Union types: Meerdere toegestane types via |, bijv. string | number.

3. Interfaces en Type Aliases

Gebruik interfaces of type aliases om de vorm van objecten te beschrijven.

  • Interfaces: Beschrijven objectstructuren, uitbreidbaar en ideaal voor data-entiteiten.
  • Voorbeeld: interface Album { id: number; title: string; }
  • Optional properties: Met ? markeer je velden die niet verplicht zijn, bijv. description?: string.

4. Functies in TypeScript

Type zowel parameters als returnwaardes om intentie en veiligheid te verhogen.

  • Parameters: function add(a: number, b: number).
  • Return type: function add(...): number voor een numeriek resultaat.
  • Void: Gebruik void als een functie niets teruggeeft.

5. TypeScript en de DOM

HTML-elementen vereisen specifieke types om de juiste properties te krijgen.

  • Type assertions: document.querySelector(...) as HTMLTemplateElement of met hoekhaak-notatie.
  • Non-null assertion (!): Gebruik ! als je zeker weet dat een element niet null is.
  • Generics bij querySelector: document.querySelector<HTMLInputElement>("#myInput") voor directe typing.

6. Events en Fetch

TypeScript heeft ingebouwde event-interfaces (zoals MouseEvent, KeyboardEvent) zodat je binnen listeners weet welke properties beschikbaar zijn.

Bij data ophalen via fetch is het best practice een interface te maken voor de JSON-structuur:

interface Album { id: number; title: string; }
fetch(url)
  .then(res => res.json() as Promise<Album[]>)

Zo weet de rest van je code dat de resultaten aan de interface voldoen.

7. Configuration (tsconfig.json)

Het tsconfig.json bestand stuurt de TypeScript compiler aan:

  • Target: Naar welke JavaScript-versie er wordt omgezet (bijv. ES6).
  • Strictness: Hoe streng de controles zijn (strict: true).
  • Include/Exclude: Welke mappen of bestanden meegenomen of genegeerd worden bij compilatie.
