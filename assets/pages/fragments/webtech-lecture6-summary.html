1. Wat is Responsive Design?

Responsive design is geen simpelweg verkleinen van elementen; het is een benadering waarbij ontwerp en ontwikkeling inspelen op gedrag en omgeving van de gebruiker (schermgrootte, platform, oriëntatie) om overal een goede UX te leveren.

  • Doel: Consistente, bruikbare ervaring onder alle omstandigheden.
  • Filosofie: "Be the browser's mentor, not its micromanager" — geef regels en hints, laat de browser beslissen op basis van het apparaat.

Responsive vs. Adaptive Design

  • Responsive Design: Eén codebase die vloeiend meebeweegt en reflowt over schermen.
  • Adaptive Design: Verschillende, op apparaatklassen geoptimaliseerde sjablonen.

2. Assen en Logical Properties

Moderne CSS gebruikt de inline- en blok-as voor richting: inline loopt met de tekststroom mee; block staat er haaks op.

  • Inline: Parallel aan de tekststroom (horizontaal voor Nederlands).
  • Block: Loodrecht op de tekststroom (verticaal voor Nederlands).

Gebruik logical properties in plaats van fysieke richtingen om te werken met verschillende schrijfrichtingen:

  • Top → block-start
  • Bottom → block-end
  • Left → inline-start
  • Right → inline-end
  • Width → inline-size
  • Height → block-size

3. De Viewport

De viewport is het zichtbare deel van het document.

Viewport Units

  • vw / vh: 1% van de breedte / hoogte van de viewport.
  • vmin / vmax: 1% van de kleinste / grootste dimensie.

Mobiele Viewports

Mobiele browsers hebben UI-elementen die in- en uitklappen; gebruik de specifieke eenheden om hierop te reageren.

  • svh / svw: Uitgeklapte UI.
  • lvh / lvw: Ingeklapte UI.
  • dvh / dvw: Dynamisch, past zich aan de huidige status aan.

De Viewport Meta Tag

Voeg altijd deze tag toe om kunstmatige verkleining op mobiel te voorkomen:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

4. Eenheden en Typografie

Absolute vs. Relatieve eenheden

  • Absolute: px, cm, mm — altijd dezelfde grootte.
  • Relatieve: em, rem, ch, % — afhankelijk van font of viewport.
  • Advies: Gebruik geen pixels voor lettertypes om gebruikersinstellingen voor leesbaarheid te respecteren.

Rem vs. Em

  • rem: Relatief aan het root-element (<html>); handig voor blok-dimensies.
  • em: Relatief aan de font-size van het element of de parent; ideaal voor inline elementen en padding die met tekst moet meeschalen.

Fluid Typography

  • Gebruik calc() en clamp(min, preferred, max) om tekst vloeiend te laten groeien binnen grenzen.
  • Modular Scale: Een harmonische reeks (bijv. gulden snede) voor consistente koppen en spacing.

5. Intrinsic Sizing

Laat elementen zichzelf dimensioneren op basis van hun inhoud.

  • min-content: Kleinst mogelijke breedte zonder overflow (vaak lengte van langste woord).
  • max-content: Breedte van de inhoud zonder tekstterugloop.
  • fit-content: Gebruikt beschikbare ruimte, maar nooit meer dan max-content.

6. Layout Technieken (Flexbox & Grid)

Flexbox

  • Eendimensionale layout (rij of kolom).
  • Items kunnen groeien (flex-grow) of krimpen (flex-shrink) om ruimte te vullen.
  • flex-basis bepaalt de initiële grootte van een item.

CSS Grid

  • Tweedimensionale layout (rijen en kolommen tegelijk).
  • Gebruik fr voor flexibele kolommen als fractie van resterende ruimte.
  • repeat(auto-fill, minmax(min, 1fr)) voor krachtige responsive grids zonder media queries; voegt kolommen toe zodra er ruimte is.

7. Media Queries

Media queries passen CSS toe in specifieke situaties.

  • Types: all, print, screen.
  • Features: width, height, orientation.
  • Breakpoints: Gebruik relatieve eenheden (rem) i.p.v. pixels; kies een breakpoint waar het ontwerp breekt, niet per apparaat.
  • Gebruik: Zie media queries als bijsturing; leun zoveel mogelijk op standaard responsive gedrag.
