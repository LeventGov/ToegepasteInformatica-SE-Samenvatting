1. Programma's vs. Processen en geheugenlayout

Programma = passief bestand; proces = actief met staat (program counter, registers, open files, geheugen).

  • Text/Code, Data, Heap (groeit omhoog, dynamic alloc), Stack (groeit omlaag, calls/locals).

2. PCB, processtaten en context switches

PCB bewaart staat (registers, PC, geheugen, I/O, accounting). Staten: New → Ready → Running → Waiting → Terminated; transities via dispatch/interrupt/I/O. Context switch = PCB A opslaan en PCB B laden → overhead zonder werk.

3. Procescreatie en relaties

  • fork(): dupliceert proces; parent krijgt child PID, child PID=0.
  • exec(): vervangt geheugen door nieuw programma.
  • wait(): parent wacht op child; zonder wait → zombie; parent dood → orphan geadopteerd door init/systemd.

4. Stack vs Heap

Stack voor aanroepen/locals, LIFO, limiet per thread; Heap voor dynamische allocaties. Pas op voor leaks en buffer overflows; pointerbeheer cruciaal (pass-by-value vs pass-by-reference).

5. Threads

Lichte processen: delen code/data/files, eigen stack/PC/registers. Voordelen: responsiviteit, resource sharing, schaalbaarheid; nadelen: race conditions en complex debuggen.

  • Applicatie beslist scheduling/aansturen (niet alleen OS).
  • Kernel threads vs user-level threads; modellen 1:1, N:1, M:N.

6. Motivatie en valkuilen

Gebruik threads voor parallel I/O (browser tabs, clients). Risico's: deadlocks, lost updates, dirty/phantom reads. Synchronisatie en juiste mapping naar OS-threads zijn cruciaal.

7. Bootproces

  • Firmware (BIOS/UEFI) → POST → kiest bootdevice.
  • Bootloader (MBR/GRUB) laadt kernel + initrd in RAM.
  • Kernel start PID 1 (systemd/init) → targets (multi-user/graphical) → services.

8. Systemd & daemons

  • Daemons: achtergrond (sshd, systemd). systemctl start/stop/restart/enable.
  • Service files: secties [Unit], [Service], [Install]; targets bepalen bootprofiel.
  • User services in ~/.config/systemd/user/, beheer met systemctl --user.
