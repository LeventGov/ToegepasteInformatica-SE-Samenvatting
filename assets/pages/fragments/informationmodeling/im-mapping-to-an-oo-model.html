Mapping to an OO model

Deze module legt de link tussen het conceptuele datamodel (ERD) en de implementatie in code, specifiek in een Object Georiënteerde (OO) taal zoals C#.

1. Van Entiteit naar Klasse

De basisregel is eenvoudig: een Entity Type in je model wordt een Class in je code.

  • Entiteit: Een concept dat uniek geïdentificeerd kan worden (bv. Member).
  • Klasse: De blauwdruk in code (public class Member).
  • Attributen: De eigenschappen van de entiteit worden velden of properties in de klasse.

Voorbeeld (C#):

public class Member(int memberNo, string name, string firstName)
{
    // Hier komen methodes en verdere logica
}

2. Attributen Mappen

Niet alle attributen zijn simpele getallen of teksten.

  • Simple Attributes: Worden direct vertaald naar types als int, string, DateOnly, etc.
  • Multivalued Attributes: Worden een lijst of collectie (bv. List<string> emailAddresses).
  • Composite Attributes: Worden ofwel "platgeslagen" (meerdere velden in de klasse) of gemapt naar een apart object/type (bv. een Address klasse).

3. Relaties Mappen

In een database werk je met Foreign Keys, maar in OO-code werk je met referenties naar objecten.

A. One-to-Many (1:N)
  • De '1'-kant (Parent): Houdt een lijst bij van kinderen.
  • Voorbeeld: Een Member heeft een lijst van Subscription objecten.
  • Code: private readonly IList<Subscription> _subscriptions;
  • De 'N'-kant (Child): Houdt een referentie bij naar de ouder.
  • Voorbeeld: Een Subscription heeft één Member eigenschap.
B. Many-to-Many (M:N)
  • In een database heb je een koppeltabel nodig. In code kan je dit directer oplossen met lijsten aan beide kanten, maar vaak wil je de logica centraliseren.
  • Facade Pattern: Soms wordt een tussenklasse gebruikt of methodes die de relatie aan beide kanten synchroon houden (AddReservation voegt het toe bij het lid én bij het spel).

4. Overerving (Inheritance)

OO-talen ondersteunen overerving direct, wat goed aansluit bij Supertype/Subtype relaties in een ERD.

  • Implementatie: De subklasse erft van de superklasse.
  • Voorbeeld: public class FamilyGame : BoardGame
  • Gevolg: De subklasse krijgt automatisch alle eigenschappen (name, publisher) van de superklasse en voegt er eigen eigenschappen aan toe (expectedGameDuration).
