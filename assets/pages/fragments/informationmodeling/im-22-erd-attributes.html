2.2 - ERD Attributes

Eigenschappen van entiteiten in een ERD: definitie, notatie en attributensoorten.

1. Wat is een attribuut?

  • Definitie: een attribuut is een functie die een entiteitsset mapt naar een waardeset.
  • Attribute-value pair: info over entiteit/relatie als attribuut-waarde paar.
  • Voorbeeld: voor Member: firstName, age, gender met waarden "Rob", "42", "male".
  • Datatypes: elk attribuut heeft een type, bijv. <string>, <integer>, <enum>.

2. Attribuut tekenen (Chen)

  • Symbool: ovaal/ellips met een lijn naar de entiteit.
  • Naamgeving: lowerCamelCase.
       ( firstName )
             |
    +-----------------+
    |     Member      |
    +-----------------+
  • Belangrijk: een attribuut verbindt aan precies een entiteit; gelijknamige attributen op verschillende entiteiten zijn conceptueel verschillend.

3. Soorten attributen

A. Optional attribute
  • Geen verplichte waarde (nullable); tegenhanger: required.
  • Voorbeeld: middleName.
B. Derived attribute
  • Waarde afgeleid/berekend uit andere attributen (soms andere entiteiten).
  • Voorbeeld: age uit birthday; totalItemPrice = quantity * unitPrice; totalAmount = som van items.
C. Multivalued attribute
  • Kan meerdere waarden per entiteit bevatten.
  • Voorbeeld: telephoneNumber (vast + mobiel).
  • Best practice: in normalisatie liever atomaire waarden; multivalued is vaak hint voor aparte entiteit (bijv. InvoiceItem).
D. Composite attribute
  • Kan opgesplitst worden in betekenisvolle delen.
  • Voorbeeld: address = street, city, postalCode, country.
  • Visueel (Chen): hoofd-attribuut aan entiteit; sub-attributen aan hoofd-attribuut.
                  ( City )
                     |
       ( Address )---+---( Street )
             |
    +-----------------+
    |     Member      |
    +-----------------+
E. Key attribute
  • Zorgt voor unieke identificatie (uitgewerkt in latere module).

4. Crow's Foot syntax (tekstueel) voor attributen

  • Optional: attribute (soms O of ?).
  • Derived: [attribute].
  • Multivalued: {attribute}.
  • Composite: (attribute1, attribute2).
  • Key: *attribute of #attribute.
