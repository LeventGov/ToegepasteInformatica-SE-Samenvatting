2.1 - ERD Entity Types

Deze module behandelt de bouwstenen van een ERD: entiteiten, entiteitssets en entiteitstypes.

1. Wat is een entiteit?

  • Definitie (Chen, 1976): een entiteit is een ding dat duidelijk geidentificeerd kan worden.
  • Voorbeelden: een specifiek persoon, bedrijf of evenement.
  • Taalkundig: in gesproken taal is een entiteit een zelfstandig naamwoord.
  • Casus: in een spelvereniging (VGC) zijn Rob, Ann en Astrid voorbeelden van leden; Mastermind en Stratego zijn voorbeelden van spellen.

2. Entity set (entiteitsverzameling)

  • Definitie: een verzameling entiteiten/instanties met gedeelde eigenschappen.
  • Gevolg: lid van set MEMBER betekent dat de entiteit de gemeenschappelijke attributen van leden heeft.
  • Verzamelingenleer (tekstueel):
(  Entity Set: MEMBER  )       (   Entity Set: GAME    )
/                        \\     /                         \\
|   Rob, Ann, Astrid       |   |  Mastermind, Stratego,    |
\\                        /     \\  Monopoly, Puzzle       /
 ------------------------       -------------------------

3. Entity type vs entiteit (instantie)

  • Entity type: het concept dat entiteiten met dezelfde eigenschappen definieert.
  • Entity instance: de concrete data/voorbeelden.
  • Voorbeeld: Rob, Ann en Astrid zijn instanties van het type Member; Mastermind en Monopoly zijn instanties van het type Game.

4. Hoe teken je een entity type?

  • Symbool: een rechthoek stelt het entiteitstype voor (niet de losse instanties).
----------------------+
|        Member        |
----------------------+
  • Naamgevingsconventie: UpperCamelCase, geen spaties of koppeltekens.
  • Regels: start met hoofdletter; elk nieuw woord ook hoofdletter; geen underscores.
  • Correct: Member, Game, SpecialMember.
  • Fout: member, GAME, special_member.
