2.5 - key attributes

Unieke identificatie binnen een entiteitsset: wat is een sleutel, waarom is context (UoD) belangrijk, en hoe noteer je sleutels.

1. Wat is een key attribute?

  • Definitie: een attribuut waarvan de waarden constant en uniek (distinct) zijn per entiteit in de set.
  • Doel: elke entiteit heeft een logische set attributen nodig om uniek te identificeren.
  • Voorbeeld: bij Car is VIN uniek; kleur of aantal deuren niet.

2. Belang van de Universe of Discourse (UoD)

  • Uniciteit hangt af van de context.
  • Scenario A (1 bedrijf): bij Wonka Industries is invoiceNumber uniek → sleutel = invoiceNumber.
  • Scenario B (accountantskantoor, meerdere bedrijven): invoiceNumber alleen is niet uniek. Nodig: combinatie invoiceNumber + company.

3. Composite key (samengestelde sleutel)

  • Definitie: combinatie van attributen die samen uniek zijn.
  • Minimaliteit: geen attribuut in de combinatie mag overbodig zijn.
  • Partial key: individueel attribuut dat deel uitmaakt van de samengestelde sleutel.
  • Voorbeeld (adres): straat + nummer + zipCode + countryCode kan samen uniek zijn.

4. Notatie (Chen)

Key attributen worden visueel onderstreept.

Voorbeeld (single key):

      ( _VIN_ )  <--- key attribute (onderstreept)
          |
    +-----------+
    |    Car    |
    +-----------+

Voorbeeld (composite key): alle deelnemende attributen (partial keys) onderstrepen.

    ( _street_ )    ( _number_ )
          \          /
           ( Address )
          /          \
   ( _zip_code_ )  ( _country_code_ )
