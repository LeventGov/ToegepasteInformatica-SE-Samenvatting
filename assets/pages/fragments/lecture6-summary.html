1. Containers vs VM's

Containers virtualiseren het OS: gedeelde kernel, geïsoleerde processen, lichtgewicht en snel te starten. VM's virtualiseren hardware via een hypervisor; elke VM heeft een volledig OS en is zwaarder maar sterker gescheiden.

2. Evolutie naar Docker

  • 1979 chroot → 2000 FreeBSD jails → 2001 Linux VServer → 2004 Solaris Zones/OpenVZ.
  • 2006 cgroups + namespaces → 2008 LXC: eerste complete Linux containers.
  • 2013 Docker open source bovenop LXC; 2014 libcontainer vervangt LXC → wordt runc.
  • 2015 OCI-standaard: Docker doneert runc en later containerd voor runtime/image specs.

3. Waarom Docker doorbrak

  • Developer ervaring: Git-achtige workflow, eenvoudige CLI, Dockerfile voor reproduceerbare builds.
  • Standaard imageformaat: build once, run anywhere, scheiding app vs infra.
  • Ecosysteem: Docker Hub voor delen, compose voor multi-container, brug tussen Dev en Ops.

4. Docker stack en alternatieven

  • Stack: Docker CLI → dockerd (Engine) → containerd (lifecycle) → runc (OCI runtime) → Linux kernel.
  • Docker Desktop: levert lichte Linux-VM + UI op Windows/macOS; stuurt dezelfde Engine aan.
  • Alternatieven runtimes: Podman (daemonless), containerd (CNCF), CRI-O (Kubernetes). Registries: Docker Hub, quay.io, images.linuxcontainers.org.

5. Docker correct installeren (Debian)

  • Vermijd verouderde distro-pakketten; gebruik Docker's repo.
  • Stapjes: apt update; installeer curl/gpg/ca-certificates/apt-transport-https.
  • Importeer sleutel → voeg repo (enkele echo-regel) → apt update → install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin.
  • Check en enable service; voeg gebruiker aan docker-groep en herlogin/newgrp; verifieer met docker --version.

6. Basiscommando's

  • docker pull (alleen downloaden); docker run (start container, evt. -d, -p).
  • docker exec -ti voor shell/commands; docker start/stop/rm voor lifecycle.
  • docker ps voor lopende containers; docker inspect voor metadata; docker logs voor output.
  • docker build -t name:tag . om eigen image te bouwen.

7. Images en tagging

  • Images zijn gelaagd (overlay2); gedeelde lagen besparen opslag. Overzicht met docker image ls.
  • Opruimen: docker image rm (niet als container gebruikt), docker image prune voor ongebruikte.
  • Tagging: gebruik semver en OS (bv. 1.27.1-alpine); vermijd latest/mainline in productie; standaardiseer tags.
  • docker image tag SOURCE[:tag] TARGET[:tag] om achteraf tags toe te voegen.

8. Networking basics

  • Standaard bridge-netwerk met docker0; containers krijgen IP binnen dit subnet.
  • Inspecteer netwerken met docker network ls/inspect; container netwerkinfo via docker inspect of ip a binnenin.
  • Publiceren via port forwarding: -p host:container (elke hostpoort maar één keer te gebruiken); iptables regelt regels.

9. Logs en lab focus

  • Logs opvragen met docker logs; opgeslagen per container in /var/lib/docker/containers//-json.log.
  • Lab: installeer via Docker repo, run hello-world, onderzoek images/containers, oefen tags, bekijk logs en bridge-netwerk/port mapping.
